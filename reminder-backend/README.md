# 邮件提醒后台设置

这个后台使用 Google Apps Script，每五分钟检查一次提醒设置。网站关闭后仍会运行，邮件通常会在设定时间后的五分钟内到达。

## 一次性设置

1. 打开 [Google Apps Script](https://script.google.com/) 并新建项目。
2. 删除编辑器中的示例代码，把本目录 `Code.gs` 的内容全部复制进去。
3. 修改文件顶部的 `CONFIG.PIN`：设置为至少 12 位、只有自己知道的随机字符串。不要把 PIN 提交到公开 GitHub 仓库。
4. 在编辑器顶部选择 `setupReminderBackend`，点击“运行”。
5. 按照 Google 提示授权脚本发送邮件和创建定时触发器。
6. 点击“部署”→“新部署”→ 类型选择“网页应用”。
7. “执行身份”选择“我”，“谁可以访问”选择“任何人”，完成部署。
8. 复制以 `/exec` 结尾的网页应用地址。
9. 打开「冲了吗」网站，右上角更多菜单选择“邮件提醒”。
10. 填入收件邮箱、提醒时间、网页应用地址和第 3 步的 PIN，然后先发送测试邮件。

## 隐私与安全

- 收件邮箱、时间和时区保存在你的 Apps Script 私有项目属性中。
- 安全 PIN 只保存在你的 Apps Script 私有代码与当前填写页面中，网站不会把 PIN 写入本地存储。
- 邮件通过你授权的 Google 账号发送，不需要把邮箱密码交给网站。
- 公开仓库里的 `Code.gs` 必须始终保留占位 PIN。

## 修改或停止

- 修改时间：回到网站的“邮件提醒”重新保存。
- 暂停邮件：关闭“启用每日提醒”后保存。
- 完全停止后台：在 Apps Script 左侧“触发器”中删除 `reminderTick`。
- 更新 Apps Script 代码后，需要重新部署一个新版本；网页应用地址通常可以继续使用原部署。

## 常见问题

- 没有收到测试邮件：检查垃圾邮件、Apps Script 执行记录、网页应用访问权限和安全 PIN。
- 邮件晚几分钟：后台每五分钟检查一次，这是正常情况。
- 修改 PIN 后无法保存：网站弹窗中的 PIN 也需要同步填写新的值。
